31/* We introduce a strong index type to differentiate between indices in the output
32 * archive (uint32_t, as used by the UpdateCallback), and the corresponding indexes
33 * in the input archive (input_index). In this way, we avoid implicit conversions
34 * between the two kinds of indices.
35 *
36 * UpdateCallback uses indices in the range [0, BitOutputArchive::itemsCount() - 1]
37 *
38 * Now, if the user doesn't delete any item in the input archive, itemsCount()
39 * is just equal to <n. of items in the input archive> + <n. of newly added items>.
40 * In this case, an input_index value is just equal to the index used by UpdateCallback.
41 *
42 * On the contrary, if the user wants to delete an item in the input archive, the value
43 * of an input_index may differ from the corresponding UpdateCallback's index.
44 *
45 * Note: given an input_index i:
46 * if i < mInputArchiveItemsCount, the item is old (old item in the input archive);
47 * if i >= mInputArchiveItemsCount, the item is new (added by the user); */
48enum class input_index : std::uint32_t {};
